发表于 2015-09-08   |   分类于 Hexo   |  

1. 思路

2. 这些要安装的都是什么东西

3. 安装 on Mac

安装brewhome.

安装nodejs

brew install node

安装hexo-用nodejs自带npm安装

npm install -g hexo
hexo init
npm install

安装git-用homebrew安装

sudo brew install git

4. 配置

多SSHkey/别名配置传送门

deploy:
type: git
repo: git@github.com:xxxx/xxxx.github.io.git
branch: master

6. 域名地址访问（域名绑定）

这里用 GoDaddy（申请域名推荐） DNSPod（域名解析推荐）

DNSpod配置

1.这里的下图被马赛克的其实就是github的IP：192.30.252.153，意思就是papp.life域名解析到github的IP，因为我们的博客放在github的仓库。

去GoDaddy里面加入DNSpod的记录

2.这边大概的意思就是去我们域名提供商那把我们域名配置用DNSpod解析，再加上上一步github的IP，这样就把域名提供商、DNSpod、github3者的关系建立起来了。(还需要第三步，在仓库加入CNAME文件)


保存

配置github

3.然后还要在我们的blog的git仓库里面放一个CNAME文件。新建一个CNAME文件里面写上域名就行了。（这里最好放在HEXO文件目录下的source文件夹，这样不会每次部署的时候被覆盖，只要配置一次就好了。）

最后等一会，一般可能要几分钟DNS循环下，然后直接访问域名就可以DNS解析到我们原本的仓库地址了
